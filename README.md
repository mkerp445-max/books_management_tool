# 蔵書管理システム (Book Management System)

施設の管理運営ルールに基づき、利用者の書籍所持数を厳格に管理するためのシステムです。

## 📋 プロジェクトの目的
本システムは、物品管理が厳しい環境において「誰が・何を・いつ取得し、いつ廃棄したか」を正確に記録し、1人あたりの所持上限（20冊）を自動でチェックすることを目的としています。

## 🛠 主な仕様
- **所持制限**: 1人につき最大20冊まで。取得時に自動計算し、上限を超える場合は登録をブロックします。
- **履歴の永続性**: 廃棄操作を行ってもデータベースからレコードを削除せず、状態を「廃棄」として保存し続けることで、監査証跡（ログ）を維持します。
- **個体管理**: 同一タイトルの書籍であっても、個別のID（BK-タイムスタンプ形式）を付与して1冊ごとに管理します。
- **論理削除**: 利用者の削除（不在設定）時もデータは物理削除せず、フラグ管理により履歴を保持します。
